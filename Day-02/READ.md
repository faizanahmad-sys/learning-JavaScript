# 📂 Day 02: Control Flow & Decision Making

Today's focus was on making JavaScript code dynamic by using Conditional Statements and understanding how the computer evaluates data.

### 📝 Key Concepts Learned:
1. **If-Else Statements:** Learned how to execute different code blocks based on conditions (e.g., age checks).
2. **Else-If Ladders:** Handling multiple conditions (e.g., Grading systems).
3. **Switch Statements:** A cleaner alternative to multiple `if` statements when dealing with specific values like days of the week.
4. **Ternary Operator:** Used for writing short, clean `if-else` logic in a single line.
5. **Truthy vs Falsy:** Understanding why `0`, `""`, and `null` behave as `false` in conditions.

### 💻 Practical Logic Implemented:
- **ATM Simulator:** Nested `if` statements to check for PIN and balance before allowing withdrawal.
- **Grading App:** Logic to assign A, B, C, or F grades based on score ranges.
- **Day Finder:** Using Switch-Case to map numbers to week days.

### 🚀 Reflection:
Conditionals are the backbone of logic. Today I learned that code isn't just a list of instructions.
